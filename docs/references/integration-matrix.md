# Integration Matrix

This matrix enumerates the external platforms Regulus relies on, outlining ownership, interface patterns, abstractions, and delivery checkpoints. Implementation teams should use it to schedule adapter work, secure access, and align with stakeholder roadmaps.

| Integration | Owning Org / Contacts | Purpose | Interface & Protocols | Regulus Abstraction | Delivery Considerations |
| --- | --- | --- | --- | --- | --- |
| Enterprise Model Inventory | Model Risk / AI Governance | Register `@ModelArtefact` agents with metadata (owner, risk tier, review cadence) | REST API (`POST /models`), OAuth2 client credentials, JSON payload aligned to inventory schema | `ModelInventoryClient` adapter bean; async publisher for change events | Capture schema version; map annotation fields to API; define retry/backoff; secure credentials via Vault |
| GRC Evidence Repository (Archer / ServiceNow GRC) | Risk & Compliance Ops | Store model cards, eval reports, approvals, change history | SFTP drop or REST API, signed payloads, TLS mutual auth | `GrceEvidenceUploader` service invoked by Gradle plugins/CI | Select transfer mode; establish file naming conventions; automate retention tags; include failure alerts |
| ServiceNow Change / Incident Mgmt | Platform Ops / SRE | Raise incidents for SEVs, record kill switch approvals | ServiceNow REST API (`/api/now/v2/table/incident` / `change_request`), OAuth + mTLS | `IncidentBridge` and `ChangeApprovalClient` triggering from kill switch and chaos tests | Define severity mapping; ensure idempotency; capture ticket IDs in audit logs |
| ConfigHub / Vault-backed Toggle Service | Enterprise Config / DevOps | Central kill switch control plane | ConfigHub REST Webhook + Spring Cloud Config bus, or Vault KV with watcher | `KillSwitchConfigAdapter` listening via Spring Cloud Config refresh | Document alternative toggle stores; support polling fallback; include dual-control metadata |
| Vendor / Outsourcing Registry | Third-Party Risk Mgmt | Track MCP/A2A/LLM providers, due diligence, exit tests | REST API or batch CSV ingest, OAuth2 + signed JWT, schema per vendor system | `VendorRegistryPublisher` invoked on connector configuration updates | Align required fields; schedule sync cadence; include error reconciliation dashboard |
| Eval / Red-Team Container (`ghcr.io/regulus/ai-evals`) | Model Safety Team | Run quality gates, jailbreak, toxicity suites | HTTPS JSON API (`/eval`, `/suite`), mutual TLS, payload redaction policy | `EvalClient` within `regulus-ai-evals-client-starter`; Gradle task `aiEvalCheck` | Define network zone; manage container deployment; document payload size limits; redact PII before send |
| Observability Stack (OTEL Collector, Grafana, Splunk) | Observability / SRE | Capture traces, metrics, audit logs | OTLP gRPC, Kafka topics (`audit.events.*`), Syslog | Auto-configured exporters and Kafka appenders | Coordinate topic naming, retention, RBAC; ensure kill switch and eval events are tagged |
| Identity & Secrets (Vault, Azure AD) | IAM / Security Engineering | Issue client credentials, rotate secrets | Vault KV v2, Azure AD app registrations, JIT tokens | Spring Vault integration; `SecretProvider` abstraction | Define secret rotation cadence; audit access; include bootstrap runbook |
| OAuth/mTLS for MCP/A2A Gateways | Security Architecture | Protect inbound/outbound interop traffic | OAuth2 client creds, mutual TLS certificates, JWKS validation | `InteropSecurityConfig` auto-config in starters | Plan cert issuance/renewal; version RBAC scopes; integrate with bank PKI |
| Data Masking Catalog | Data Privacy Office | Maintain JSONPath selectors and minimisation rules | Git-backed catalog or REST fetch; signed config bundle | `PrivacyFilterConfigLoader` reading from central source | Define approval workflow for rule changes; version guards; support regression tests |

> Update this matrix as adapters move from prototype to production to keep integration owners aligned. For new institutions, clone the table and capture bank-specific systems before kickoff.
